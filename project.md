# project.md

## Cengage Listening Activity Solver (Playwright Automation)

### Overview

This project automates **Cengage MindTap listening practice activities** that are launched through **Brightspace (Purdue LMS)**.

The solver:

1. Opens the **Brightspace course activity page**
2. Clicks **“Open in New Window”** to launch the MindTap assignment
3. Waits for the MindTap reader to load
4. Expands the listening activity
5. Clicks **Start/Resume Assignment**
6. Iterates through each question
7. Selects answers until the correct one is detected
8. Moves to the next question
9. Continues until the activity is complete

The automation is implemented using **Playwright (Python)**.

---

# System Architecture

```
Brightspace (LMS)
      ↓
Open in New Window
      ↓
MindTap Reader Page
      ↓
Main Reader iframe (#1_NB_Main_IFrame)
      ↓
Activity Container
      ↓
MindTap Activity iframe (mindapp_iframe)
      ↓
Quiz Interface
```

Automation must navigate through **two nested iframes** to reach the quiz.

```
.check-my-work-link
```

Two identical buttons exist:

```

```

Solution:

```
frame.locator(".check-my-work-link").first.click()
```


# Quiz Solving Algorithm

Pseudo-logic:

```
for each question:

    for each answer choice:
        click answer
        click "Check My Work"
        read feedback

        if feedback != "Incorrect":
            break

    click "Next"
```

The script brute-forces answers until the correct one is found.

# Brightspace Launch Problem

Originally the script attempted to open:

```
https://ng.cengage.com/static/nb/ui/evo/index.html...
```

This failed with:

```
/static/nb/logout.html
```

Reason:

MindTap requires an **LTI launch token generated by Brightspace**.

Directly opening the Cengage link bypasses this authentication.

---

# Solution

Instead of loading Cengage directly:

```
Brightspace page
↓
click "Open in New Window"
↓
MindTap launches with valid token
```

Playwright now reproduces the correct LMS launch flow.

# Brightspace Login Issue

Brightspace frequently redirected to:

```
sessionExpired=0
```

because authentication used:

* Purdue SSO
* Duo 2FA
* short-lived cookies

---

# Solution: Save Login Session

We use Playwright's **storage_state** to capture cookies after login.

## Login Capture Script

```
save_login.py
```

Steps:

1. Open Brightspace
2. Login manually with SSO + 2FA
3. Save session

Output file:

```
brightspace_auth.json
```

---

# Solver Script Behavior

The solver loads the saved session:

```
browser.new_context(storage_state="brightspace_auth.json")
```

This bypasses login for future runs.

# Problems Encountered and Fixes

## 1. CSS selector invalid

Error:

```
'#1_NB_Main_IFrame' not valid selector
```

Cause:

CSS IDs cannot start with numbers.

Fix:

```
iframe[id="1_NB_Main_IFrame"]
```

---

## 2. Duplicate elements (strict mode)

Playwright strict mode requires exactly one match.

Errors occurred for:

```
.check-my-work-link
img[alt="Next"]
```

Fix:

```
.first.click()
```

---

## 3. Quiz answers not detected

Cause:

Quiz iframe had not loaded yet.

Fix:

```
frame.locator("input[type='radio']").first.wait_for()
```

---

## 4. MindTap iframe not loaded

Activity initially collapsed:

```
display:none
src=""
```

Fix:

Click expand button before accessing iframe.

---

## 5. Direct Cengage link logout

Cause:

Missing Brightspace LTI token.

Fix:

Launch activity through Brightspace.
